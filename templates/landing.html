Enterprise Capabilities

Built to catch everything.

Our engine goes far beyond simple syntax checking. We render code exactly how a browser does to ensure structural and visual perfection.

DOM Assertions

Check if specific elements exist, count nodes, and verify tree structures instantly using standard CSS selectors.

Computed Styles

We don't parse strings. We evaluate `window.getComputedStyle` to ensure layouts (like Flexbox and Grid) actually resolve correctly.

Visual Heatmaps

Pixelmatch algorithms generate a red-highlighted diff image, comparing the student's UI against the trainer's baseline.

Interaction Triggers

Simulate user behavior. Puppeteer types into inputs, clicks buttons, and waits for DOM updates to verify JS logic.

Secure Sandbox

Complete network interception. We block unauthorized external APIs and force strict CPU/memory limits to prevent infinite loops.

Partial Scoring

JSON rubrics assign weighted marks. A 2% pixel difference might yield 80% marks, while a missing button fails a 5-mark DOM test.

How PRISM Works

A seamless pipeline from question creation to instant grading.

1

Trainer JSON

Trainers define rubrics, viewports, and specific DOM/CSS tests in a JSON spec.

2

Baseline Master

PRISM renders the master solution and saves perfect "Golden Standard" screenshots.

3

Student Codes

Students build the UI in their portal. Code is injected into our Node engine on submit.

4

Engine Execution

DOM is tested, visuals are diffed, and a detailed JSON report with scores is saved.
